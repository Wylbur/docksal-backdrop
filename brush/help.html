Command detail

archive-dump
Backup your code, files, and database into a single file.

Examples:
 brush archive-dump                         Write an archive containing 3 sites in it.
 default,example.com,foo.com
 brush archive-dump @sites                  Save archive containing all sites in a multi-site.
 brush archive-dump default                 Save archive to custom location.
 --destination=/backups/mysite.tar


Arguments:
 targets                                   Optional. Site specifications, delimited by commas. Typically, list subdirectory name(s) under
                                           /sites.


Options:
 --description                             Describe the archive contents.
 --tags                                    Add tags to the archive manifest. Delimit multiple by commas.
 --destination                             The full path and filename in which the archive should be stored. If omitted, it will be saved
                                           to the brush-backups directory and a filename will be generated.
 --overwrite                               Do not fail if the destination file exists; overwrite it instead.
 --generator                               The generator name to store in the MANIFEST file. The default is "Brush archive-dump".
 --generatorversion                        The generator version number to store in the MANIFEST file. The default is 1.0-dev.
 --pipe                                    Only print the destination of the archive. Useful for scripts that don't pass --destination.
 --preserve-symlinks                       Preserve symbolic links.


Aliases: ard, archive-backup, arb

site-upgrade
Execute a major version upgrade for Backdrop core and enabled contrib modules. Command will download next version of Backdrop and all
available contrib modules that have releases. It prepares a settings.php for the target site, and copies the prior version's database to the
target site. Finally, updatedb is run. The intent is for developers to keep re-running this command until they are satisfied with the
resulting site. Run this command from within your source site (D6). Note that this command uses pm-download and sql-sync internally so most
options for those commands are valid here too.

Examples:
 brush site-upgrade @onward                 Upgrade from the current site to the site specified by @onward alias.


Arguments:
 target                                    The name of a sitealias, which points to the destination site. 'root' and 'uri' keys are
                                           required; db-url is recommended. See examples/aliases.brushrc.php for more information about
                                           creating a site alias.


Options:
 --structure-tables-key                    A key in the structure-tables array. @see example.brushrc.php. Defaults to 'common'.
 --source-dump                             Path to dump file. Medium or large sized sites should set this. Optional; default is to create a
                                           temporary file.
 --db-su                                   DB username to use when dropping and creating the target database. Optional.
 --db-su-pw                                DB password to use when dropping and creating the target database. Optional.
 --no-cache                                Transfer a fresh database from source site. Otherwise, DB dump is re-used for 24 hours.
 --core-only                               Stop after upgrading Backdrop core; do not download and enable new versions of the site's
                                           modules.
 --force-sites-default                     Forces settings.php to be written in sites/default folder, even if source settings.php is not.
 --replace                                 Replace target if it already exists.  Default is to prompt.
 --reuse                                   Reuse target if it already exists.  Default is to prompt.
 --uninstall                               Comma-separated list of modules to uninstall in the target database prior to upgrade.  n.b. The
                                           source site is not affected.

pm-download
Download Backdrop core or projects from drupal.org (Backdrop core, modules, themes or profiles) and other sources. It will automatically
figure out which project version you want based on its recommended release, or you may specify a particular version.

If no --destination is provided, then destination depends on the project type:
  - Profiles will be downloaded to profiles/ in your Backdrop root.
  - Modules and themes will be downloaded to the site specific directory (sites/example.com/modules|themes) if available, or to
sites/all/modules|themes.
  - If you're downloading backdrop core or you are not running the command within a bootstrapped backdrop site, the default location is the
current directory.
  - Brush commands will be relocated to /usr/share/brush/commands (if available) or ~/.brush. Relocation is determined once the project is
downloaded by examining its content. Note you can provide your own function in a commandfile to determine the relocation of any project.

Examples:
 brush dl                                   Download latest recommended release of Backdrop core.
 brush dl backdrop                          Same as `brush dl`.
 brush dl cck zen                           Download latest versions of CCK and Zen projects.
 brush dl og-1.3                            Download a specfic version of Organic groups module for my version of Backdrop.
 brush dl diff-6.x-2.x                      Download a specific development branch of diff module for a specific Backdrop version.
 brush dl views --select                    Show a list of recent releases of the views project, prompt for which one to download.
 brush dl webform --dev                     Download the latest dev release of webform.
 brush dl cck --version-control=bzr         Download the cck project and then add it and commit it to Bazaar.
 --bzrsync --bzrcommit
 brush pm-download cck                      Commit changes as the user 'joe' (Quotes are required).
 --svncommitparams=\"--username joe\"
 brush pm-download cck                      Checkout should use these credentials.
 --cvscredentials=\"name:password\"
 brush pm-download cck --cvsparams=\"-C\"   Overwrite all local changes (Quotes are required).
 brush pm-download cck --cvsmethod=update   Will update the project, and try to merge changes, rather than overwriting them. Any conflicts
                                           will need to be resolved manually.
